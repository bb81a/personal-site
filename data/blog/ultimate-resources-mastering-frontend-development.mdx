## Back to the Roots - JavaScript and Engines

### JustJavaScript.com (Paid)

![](/static/images/ultimate-resources-mastering-frontend-development/justjavascript.png)

By React wizard Dan Abramov, this course presents JavaScript mental models in ten episodes. It could have been free if you'd time-traveled a few years back! It's a fascinating journey across JavaScript's universe, shedding light on:

- Values & variables
- Statements & expressions
- Primitive types
- Objects & functions
- null & undefined
- Properties & prototypes

Check it out: [https://justjavascript.com/](https://justjavascript.com/)

### JavaScript visualized (Free)

![](/static/images/ultimate-resources-mastering-frontend-development/jsvisualized.gif)

Lydia Hallie wrote dev.to articles elucidating complex concepts most developers would rather avoid. Super fun to read. These articles are seriously some of the best.

Dive into these seven topics:

- [Promises & Async/Await](https://dev.to/lydiahallie/javascript-visualized-promises-async-await-5gke)

## React

I'm a strong advocate of [React's documentation](https://react.dev/learn) as the prime tutorial. The 2023 version is stellar and can be consumed over a laid-back weekend. Discover some hooks that were hidden till now. However, if you want a glitzier learning experience, like video tutorials or an interactive playground, here are two I've tried and loved:

### EpicReact.dev (Paid)

![](/static/images/ultimate-resources-mastering-frontend-development/epicreact.png)

A vintage 2020 course by Kent C. Ddods that gives you a backstage pass to React. Kent demystifies babel, virtualization, breaks down each hook with fantastic examples (in public GitHub repos). He illustrates React patterns, like Compound Components, and eventually constructs a full application.

Have a look: [https://epicreact.dev/](https://epicreact.dev/)

### React Visualized (Free'ish)

![](/static/images/ultimate-resources-mastering-frontend-development/reactvisualized.png)

This resource has everything about React presented in a fun, interactive manner. They also have a paid course for video buffs. The authors are well-known in the React ecosystem, curate one of the biggest newsletters on React/JavaScript, and have created many courses on React Query, TypeScript, React Router, etc.

Check it out: [https://react.gg/visualized](https://react.gg/visualized)

## TypeScript

Okay, be honest, the TypeScript documentation could be friendlier, right? It's like a maze, making it challenging to grasp. But, here comes our hero, Matt Pocock.

### Total TypeScript (Paid)

![](/static/images/ultimate-resources-mastering-frontend-development/totaltypescript.png)

If Matt's YouTube content brought a smile to your face, then this course is a must. He has numerous free blog posts on popular topics, like React.ReactNode vs JSX.Element. Plus, a plethora of free workshops, including React with TypeScript and Zod tutorials.

Visit: [https://www.totaltypescript.com/](https://www.totaltypescript.com/)

## Architecture

### Patterns.dev (Free)

![](/static/images/ultimate-resources-mastering-frontend-development/patternsdev.png)

Lydia Hallie and Addie Osmani bring you another resource, especially useful for React developers. It covers more JavaScript patterns than you thought existed. They even highlight performance patterns and concepts like Hydration. It may need more than a weekend unless you're popping adderall daily üòÜ

Link: [https://patterns.dev/](https://patterns.dev/)

### Testing JavaScript

![](/static/images/ultimate-resources-mastering-frontend-development/kenttesting.png)

Kent C. Ddods returns with a still relevant course, despite its age. It covers all the frontend testing techniques you need to know. Unit testing, integration testing, and E2E. It doesn't touch TDD or practices like `AAA`, `GWT` (but I guess that's more a Java geek thing ü§ì).

Link: [https://testingjavascript.com/](https://testingjavascript.com/)

### JavaScript/React Patterns

![](/static/images/ultimate-resources-mastering-frontend-development/jspatterns.png)

Lydia is back with a resource similar to patterns.dev. The patterns discussed here and in the workshop can guide you when you confront problems other developers have faced before. It's not a one-size-fits-all solution, but rather an awareness-raising tool for specific patterns, problems they solve, and their implementation.

Link: [https://javascriptpatterns.vercel.app/patterns](https://javascriptpatterns.vercel.app/patterns)

### Clean Coders

![](/static/images/ultimate-resources-mastering-frontend-development/unclebob.jpg)

Uncle Bob (Robert C. Martin) might not ring a bell for many modern frontend developers, or you might not agree with his views. Yes, times have changed, and we're more focused on making something that "kind of works now" instead of a product that's "great and will stay great". But there's a treasure trove of info here that will take you a few months to consume (not joking, each video is approx. 1 hour long + workshops/homework).

- Starting from basics like `SOLID` (applicable to non-OOP world)
- Architecture concepts and UML galore - `component cohesion & coupling`
- Functional programming
- Design patterns
- Advanced `TDD` (you might need to invest in a tri-color (fail üü•, refactor üîÅ, pass üü©) cap üß¢)
- `BDD`

Link: [https://cleancoders.com/](https://cleancoders.com/)
